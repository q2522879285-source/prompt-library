# 质量评估矩阵

## 评估维度

### 1. 清晰度 (Clarity) - 权重 25%
```
评估标准:
5分: 语言明确无歧义，表述精准
4分: 基本清晰，偶有模糊表达
3分: 部分表述不够清晰
2分: 存在明显歧义
1分: 表述混乱难以理解

评估要点:
- 用词是否精确
- 逻辑是否通顺
- 是否存在歧义表达
- 是否有冗余内容
```

### 2. 完整性 (Completeness) - 权重 25%
```
评估标准:
5分: 6要素齐全，结构完整
4分: 缺失1个次要要素
3分: 缺失1个重要要素或2个次要要素
2分: 缺失多个重要要素
1分: 结构严重残缺

评估要点:
- 角色定义是否完整
- 任务描述是否清晰
- 约束条件是否充分
- 输出格式是否明确
```

### 3. 可执行性 (Executability) - 权重 30%
```
评估标准:
5分: AI可直接执行，无需额外理解
4分: 基本可执行，需少量推理
3分: 需要一定推理才能执行
2分: 执行存在困难
1分: 难以直接执行

评估要点:
- 指令是否具体可操作
- 是否有明确步骤
- 是否有决策依据
- 是否有异常处理
```

### 4. 鲁棒性 (Robustness) - 权重 10%
```
评估标准:
5分: 考虑各种边界情况
4分: 考虑主要边界情况
3分: 有一定边界处理
2分: 边界处理不足
1分: 无边界考虑

评估要点:
- 异常输入处理
- 边界情况覆盖
- 错误恢复机制
- 容错能力
```

### 5. 创新性 (Innovation) - 权重 10%
```
评估标准:
5分: 有独特创新技巧
4分: 有技巧组合创新
3分: 使用了常见技巧
2分: 技巧使用较少
1分: 无明显技巧

评估要点:
- 是否有创新技巧
- 技巧组合是否新颖
- 是否有独特视角
- 是否有原创方法
```

---

## 计算公式

```
总分 = 清晰度×0.25 + 完整性×0.25 + 可执行×0.3 + 鲁棒性×0.1 + 创新性×0.1
```

## 评级标准

| 评级 | 分数范围 | 处理方式 |
|------|---------|---------|
| S级 | 4.5 - 5.0 | 直接应用到系统 |
| A级 | 4.0 - 4.4 | 建议应用，可微调 |
| B级 | 3.5 - 3.9 | 收录参考，适用时应用 |
| C级 | 3.0 - 3.4 | 收录，需改进后再考虑 |
| D级 | < 3.0 | 不收录 |
