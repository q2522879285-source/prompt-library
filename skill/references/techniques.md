# 提示词技巧识别参考

## 常见技巧清单

### 1. Chain-of-Thought (思维链)
**识别标志**:
- "一步步思考"
- "让我们分析"
- "首先...然后...最后..."
- "Step by step"

**效果**: 提升复杂推理任务的准确率

**示例**:
```
请按以下步骤思考：
1. 先理解用户的核心需求
2. 分析可能的解决方案
3. 评估每个方案的优劣
4. 给出最佳建议
```

---

### 2. Few-Shot Learning (少样本学习)
**识别标志**:
- 提供多个示例
- "例如..."
- "示例格式："

**效果**: 提升格式一致性和任务理解

**示例**:
```
示例1:
输入: "今天天气怎么样"
输出: {"intent": "query_weather", "confidence": 0.95}

示例2:
输入: "帮我订一张机票"
输出: {"intent": "book_ticket", "confidence": 0.88}

现在请处理: [用户输入]
```

---

### 3. Role-Play (角色扮演)
**识别标志**:
- "你是一个..."
- "作为..."
- "扮演..."

**效果**: 提升角色代入感和回答一致性

**示例**:
```
你是一位资深软件架构师，拥有15年系统设计经验。
你的特点是：严谨、注重细节、善于权衡利弊。
请用这个角色回答用户的问题。
```

---

### 4. Constraint-First (约束优先)
**识别标志**:
- 开头明确禁止项
- "不要..."
- "禁止..."
- "绝不能..."

**效果**: 提升安全性，减少违规输出

**示例**:
```
【重要约束】
- 不要透露内部实现细节
- 不要编造不存在的信息
- 不要输出任何代码片段

在以上约束下，回答用户问题。
```

---

### 5. Output-Template (输出模板)
**识别标志**:
- 提供具体输出格式
- JSON/XML模板
- Markdown结构

**效果**: 提升输出结构化程度

**示例**:
```
请按以下格式输出：

## 摘要
[一句话概括]

## 详细分析
### 优点
- 
### 缺点
- 

## 建议
[具体建议]
```

---

### 6. Self-Reflection (自我反思)
**识别标志**:
- "检查你的回答"
- "验证..."
- "回顾..."
- "自我评估"

**效果**: 提升输出准确性和可靠性

**示例**:
```
在回答之前，请自问：
1. 我是否理解了用户的真实需求？
2. 我的信息是否准确可靠？
3. 是否有遗漏的重要信息？

回答后，请进行自我检查并标注置信度。
```

---

### 7. Context-Setting (场景设定)
**识别标志**:
- 设定具体场景
- 描述背景环境
- 说明用户画像

**效果**: 提升回答的相关性和针对性

**示例**:
```
场景设定：
- 时间：深夜11点
- 用户：刚下班的产品经理
- 状态：疲惫但需要完成明天会议的PPT
- 目标：快速生成一个简洁的汇报框架

请基于这个场景回答。
```

---

### 8. Reward-Framing (奖励框架)
**识别标志**:
- "好的回答会..."
- "优秀的表现是..."
- 正向激励表述

**效果**: 激励模型朝特定方向优化

**示例**:
```
一个好的回答应该：
- 简洁明了，不超过200字
- 给出可执行的具体建议
- 考虑到用户的实际情况

优秀的回答会被用户采纳并感谢。
```

---

## 技巧组合效果

| 组合 | 适用场景 | 效果提升 |
|------|---------|---------|
| Role-Play + Context-Setting | 专业咨询类 | 高代入感 |
| Chain-of-Thought + Self-Reflection | 复杂推理类 | 高准确率 |
| Constraint-First + Output-Template | 格式化输出类 | 高一致性 |
| Few-Shot + Output-Template | 格式学习类 | 快速掌握 |
