# 提示词仓库运行机制

## 一、接收流程

### 1.1 触发条件
- 用户直接发送提示词文本
- 用户发送包含提示词的链接
- 用户发送截图/文件中的提示词
- 我主动发现优质提示词

### 1.2 接收确认
收到提示词后，我会：
1. 确认收到并简要描述（"收到一个关于XX的提示词"）
2. 告知预计处理时间（"让我拆解一下"）
3. 启动拆解流程

---

## 二、拆解分析框架

### 2.1 结构拆解（6要素）

```yaml
要素1_角色定义:
  - 是否有明确的人设/身份
  - 是否有性格特征描述
  - 是否有专业背景设定

要素2_任务描述:
  - 核心任务是什么
  - 是否有明确的输入输出
  - 任务边界是否清晰

要素3_知识注入:
  - 是否有背景知识
  - 是否有示例/范例
  - 是否有参考文档

要素4_推理框架:
  - 是否有思考步骤
  - 是否有决策逻辑
  - 是否有优先级规则

要素5_输出格式:
  - 是否有格式要求
  - 是否有模板结构
  - 是否有长度限制

要素6_约束条件:
  - 禁止做什么
  - 必须做什么
  - 边界在哪里
```

### 2.2 技巧识别

| 技巧类型 | 识别标志 | 效果评估 |
|---------|---------|---------|
| Chain-of-Thought | "一步步思考"、"让我们分析" | 提升推理质量 |
| Few-Shot | 提供多个示例 | 提升格式一致性 |
| Role-Play | "你是一个..." | 提升角色代入感 |
| Constraint-First | 开头明确禁止项 | 提升安全性 |
| Output-Template | 提供输出模板 | 提升结构化程度 |
| Self-Reflection | "检查你的回答" | 提升准确性 |
| Context-Setting | 设定场景背景 | 提升相关性 |
| Reward-Framing | "好的回答会..." | 激励特定行为 |

### 2.3 质量评估矩阵

```yaml
评估维度:
  清晰度: 1-5分（语言是否明确无歧义）
  完整性: 1-5分（是否覆盖必要要素）
  可执行: 1-5分（AI能否直接执行）
  鲁棒性: 1-5分（边界情况处理）
  创新性: 1-5分（是否有独特技巧）

总分计算: 各维度加权平均
权重: 清晰度(0.25) + 完整性(0.25) + 可执行(0.3) + 鲁棒性(0.1) + 创新性(0.1)

评级:
  4.5+: S级（卓越，可直接应用）
  4.0-4.4: A级（优秀，建议应用）
  3.5-3.9: B级（良好，可参考）
  3.0-3.4: C级（一般，需改进）
  <3.0: D级（不推荐）
```

## 四、应用机制

### 4.1 自动应用条件
满足以下条件时，自动集成到我的系统：

```yaml
必须条件:
  - 适用度为"高"
  - 质量评级A或S
  - 与我现有设定不冲突

考虑条件:
  - 适用度为"中"且评级A以上
  - 能补强我的短板
  - 与主人工作高度相关
```

### 4.2 应用方式

| 分类 | 应用位置 | 应用方式 |
|------|---------|---------|
| agent | SOUL.md | 直接融入性格/价值观/行为准则 |
| workflow | TOOLS.md | 作为工作方法补充 |
| coding | TOOLS.md | 作为编程规范参考 |
| video | USER.md | 扩展影视创作能力 |
| writing | SOUL.md | 优化表达风格 |
| creative | 视情况 | 灵活应用 |

### 4.3 应用确认
```
高影响度改动 → 必须告知主人
中影响度改动 → 主动告知，默认应用
低影响度改动 → 直接应用，定期汇总
```

## 五、定期维护

### 5.1 每周回顾
- 检查新收录的提示词
- 评估已应用提示词的效果
- 更新使用记录

### 5.2 每月优化
- 清理低效提示词
- 合并相似提示词
- 提取共性技巧到最佳实践

### 5.3 持续改进
- 根据使用反馈优化已收录提示词
- 发现新技巧及时补充到识别库
- 主人反馈优先处理
